== Risks and Technical Debts

The main current risk is no longer a missing extension boundary. It is drift between the real extension entrypoint, the converter’s actual coverage, and the way docs or package examples describe that coverage.

[options="header",cols="1,3"]
|===
|Priority |Risk and mitigation

|1
|The repository now ships a real extension entrypoint as well as the markdown kernel. The main risk is drift between the extension entrypoint, the converter’s real coverage, and the way docs or package examples describe that coverage.

Mitigation:

* keep the real extension entrypoint, README examples, and release checks aligned
* remove dead scaffolding artifacts when they linger in docs or tests
* treat repository self-consistency checks as release gates
* extend converter coverage and tests together whenever richer assembled AsciiDoc constructs are supported
|===
