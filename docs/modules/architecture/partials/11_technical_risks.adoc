== Risks and Technical Debts

The main currently documented risk is the one described by the note `Scaffold leftovers must be removed before contract expansion`.

Priority 1::

The repository’s markdown kernel and validation surfaces are much more mature than its Antora integration boundary. If documentation, package examples, or public entrypoints imply fully implemented Antora registration before it exists, maintainers will inherit false confidence, noisy reviews, and avoidable churn.

Mitigation::

* keep scaffold naming explicit in `src/extension/index.ts` and public docs
* remove dead scaffold artifacts before broadening public contracts
* treat repository self-consistency checks as release gates
* implement the real Antora registration slice before renaming scaffold boundaries to stronger terms
